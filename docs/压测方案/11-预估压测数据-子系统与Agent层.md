#### 1.1.3 Provider 容灾切换预估

**熔断器状态机转换时序**:

```
Timeline (秒)
0s────5s────10s────15s────...────60s────65s────70s────75s
 │    │      │      │            │      │      │      │
 ▼    ▼      ▼      ▼            ▼      ▼      ▼      ▼
 F1   F2     F3     F4(50%触发)   F5     探测1  探测2  恢复
                     │            │      │      │
                     OPEN ────────┼──── HALF_OPEN ── CLOSED
                     │            │      │
                     切换fallback   60s 冷却   5个探测请求
```

**详细时序预估**:

| 事件 | 时间点 | 延迟 | 状态 | 说明 |
|------|--------|------|------|------|
| 正常请求 | T+0 | ~2s | CLOSED | 主 Provider 正常 |
| 第 1 次失败 | T+1 | ~60s (timeout) | CLOSED | 计数 +1 |
| 第 2 次失败 | T+2 | ~60s | CLOSED | 计数 +1 |
| 第 3 次失败 | T+3 | ~60s | CLOSED | 计数 +1 |
| 第 4 次失败 | T+4 | ~60s | CLOSED → OPEN | 50% 失败率触发 |
| 切换到 fallback | T+4 | < 1ms | OPEN | 立即生效 |
| fallback 请求 | T+5 | ~3s | OPEN | 使用 Claude |
| 冷却等待 | T+5~T+64 | — | OPEN | 60s 冷却期 |
| 探测请求 1 | T+65 | ~2s | HALF_OPEN | 自动探测 |
| 探测请求 2-5 | T+66~T+69 | ~2s each | HALF_OPEN | 连续成功 |
| 恢复 | T+70 | — | CLOSED | 切回主 Provider |

**容灾切换期间延迟影响**:

| 阶段 | 使用 Provider | 预估延迟 P50 | 预估延迟 P95 | 额外开销 |
|------|-------------|-------------|-------------|---------|
| 正常 (CLOSED) | GPT-4o | ~2.5s | ~5s | 0 |
| 切换中 (最后一次失败) | GPT-4o → timeout | ~60s | ~60s | +58s |
| 降级运行 (OPEN) | Claude-3.5-Sonnet | ~2s | ~4s | -0.5s（更快） |
| 探测期 (HALF_OPEN) | GPT-4o (少量) | ~2.5s | ~5s | 0 |
| 恢复 (CLOSED) | GPT-4o | ~2.5s | ~5s | 0 |

**关键指标**: 从检测到故障到完成切换约 4 次失败 × 60s 超时 = **最长 4 分钟**。

### 3.5 Worker/进程模型预估

**uvicorn 配置预估**:

| 配置 | 值 | 预估效果 |
|------|---|---------|
| workers=1 (默认) | 单进程 | QPS ~10 (诊断), ~8000 (健康) |
| workers=2 | 双进程 | QPS ~18 (诊断), ~15000 (健康) |
| workers=4 | 四进程 | QPS ~30 (诊断), ~25000 (健康) |
| workers=4 + uvloop | 四进程 + 快速事件循环 | QPS ~36 (诊断), ~30000 (健康) |

**多 Worker 内存影响**:

| Workers | 基线内存 | 峰值内存(10QPS) | BGE-M3 共享 | 说明 |
|---------|---------|----------------|------------|------|
| 1 | ~276MB | ~500MB | 独占 200MB | 默认 |
| 2 | ~476MB | ~900MB | 各 200MB ⚠️ | 模型不共享 |
| 4 | ~876MB | ~1.8GB ⚠️ | 各 200MB | 需要更多内存 |

**结论**: 多 Worker 模式下 BGE-M3 模型每个 Worker 独占 ~200MB，4 Worker 峰值内存接近 2GB。建议使用独立 embedding 服务避免重复加载。
